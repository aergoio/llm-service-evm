The Protocol

How It Works

A fully on-chain oracle pattern. Your contract sends a request, authorized nodes race to fulfill it, and the result is delivered back via callback.

01

Contract sends request

Your contract calls newRequest() with a prompt config hash, JSON input, and the name of the callback function to invoke when done.

On-chain
02

Nodes process with AI

Authorized off-chain nodes detect the NewRequest event, fetch the prompt config, inject your JSON inputs, and call the AI provider API.

Off-chain
03

Result delivered via callback

Once the required number of nodes agree (quorum), the contract triggers your callback function with the AI-generated result — fully on-chain.

On-chain
Quorum-based redundancy. Set redundancy > 1 to require multiple independent nodes to return the same result before your callback fires. Eliminate unreliable nodes without changing your contract.

Why HPP AI Service

Everything your contract needs

Quorum Consensus

Set redundancy to require N independent nodes to return identical results. Unreliable or colluding nodes can't corrupt your callback.

Off-chain Prompt Storage

Store your prompt config off-chain and reference it by SHA256 hash. Update prompts without redeploying — and keep sensitive instructions private.

Dynamic JSON Inputs

Pass any runtime data as a JSON string. Use {"{{"}key{{"}}"}} placeholders in your prompt template that get substituted per-request — perfect for user-driven AI interactions.

Async Callback Pattern

No polling. Your contract receives results asynchronously via a named callback function. Design around events naturally, just like how HTTP webhooks work.

Result Tag Extraction

Enable returnContentWithinResultTag to let nodes automatically extract only the content inside <result> tags — keeping AI responses structured and predictable.

Off-chain Result Storage

For large responses, enable storeResultOffchain — nodes store the output and return only the hash on-chain. Your contract gets a verifiable reference, not a gas-heavy string.

Integration

Three steps to AI-powered contracts

Implement the callback interface, send a request with your prompt config hash, and handle the result — that's it.

1

Implement the callback interface

Inherit ILLMServiceCallback and declare your handler function.

2

Send an AI request

Call newRequest() with your config hash, JSON inputs, and callback name.

3

Handle the result

Receive the AI-generated result string inside your callback and act on it.

4

Prompt config format

Store your prompt off-chain. First line sets the model; use {"{{key}}"}} for runtime values.

MyContract.sol 
import "./llm-service.sol";

contract MyContract is ILLMServiceCallback {
    ILLMService public aiService;

    constructor(address _aiService) {
        aiService = ILLMService(_aiService);
    }
}
Contract Interface 
interface ILLMService {
    function newRequest(
        bytes32 platform,           // AI platform ("openai", "anthropic", …)
        bytes32 model,              // Model identifier (or use config)
        bytes32 configHash,         // SHA256 of off-chain prompt config
        string  input,              // JSON string with runtime values
        uint8   redundancy,         // Nodes that must agree (quorum)
        bool    extractResultTag,    // Extract content from <result> tags
        bool    storeResultOffchain, // Store large responses off-chain
        string  callbackFunction,    // Name of your callback method
        bytes   args                 // Extra args forwarded to callback
    ) external payable returns (uint256 requestId);
}

interface ILLMServiceCallback {
    function handleResult(
        uint256 requestId,
        string  calldata result,
        bytes   calldata args
    ) external;
}

Decentralisation

Run an AI Service Node

Node operators power the network by listening for requests, calling AI provider APIs, and submitting results back to the contract. Bring your own API keys.

Clone & install

Clone the repo and install Node.js dependencies.

Configure API keys

Add your AI provider API keys to the node config. Support any subset of platforms.

Run & authorize

Start the node — it auto-generates a wallet. Fund it with gas and apply to be authorized on the contract.

terminal 
# Clone & install
git clone https://github.com/aergoio/llm-service-evm
cd llm-service-evm/nodes
npm install

# Run against HPP Sepolia testnet
node llm-service.js hpp-sepolia

# Or keep it alive with pm2
pm2 start llm-service.js \
    --name "ai-service-sepolia" \
    -- hpp-sepolia
